# KiviCare Bulk Import (Slice 8d) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A fully-functional synchronous bulk import at `/api/v1/import` for 9 entity types (taxes, services, clinics, appointments, encounters, prescriptions, medical-history, doctors, patients) — real CSV + JSON parsing, per-row validation, conflict strategies, dry-run, and WP-user provisioning for doctors/patients.

**Architecture:** A generic import engine + a registry of per-entity adapters. Each adapter declares a Zod row schema, a conflict key, and an `insertRow` that reuses an existing KC create-path where one exists (taxes, encounters, prescriptions, medical-history) or does a parameterized raw INSERT (clinics, services, appointments) or WP provisioning (doctors, patients). Processing is **synchronous, per-row** (collect errors, continue) — matching the design's "synchronous, no background jobs" statement. The `jobId` in the response is a correlation id, not a persisted row (no import-jobs table exists and none is added).

**Tech Stack:** Next.js 14 App Router, TypeScript strict, Prisma 5 + MySQL (raw SQL), Zod, `papaparse` (added), Vitest.

**Branch:** `feat/kc-advanced` (already created from `main`).

## Endpoints (synchronous model)
| Method | Path | Capability | Behavior |
|--------|------|-----------|----------|
| POST | `/api/v1/import` | `import_manage` | import rows for one entity (JSON body or multipart CSV) → summary |
| POST | `/api/v1/import/validate` | `import_manage` | dry-run: parse + validate, no writes |
| GET | `/api/v1/import/templates` | `import_manage` | column templates + sample rows (all entities, or `?entity=`) |

> Note: the design's `/import/jobs`, `/jobs/{id}`, `/jobs/{id}/cancel` are omitted — they presuppose background job processing, which the design explicitly excludes ("processing is synchronous"). A synchronous import returns its full result inline, so job polling/cancel have no meaning. This is documented in the PR.

## Request shape
```
POST /api/v1/import
  Content-Type: application/json
  { "entity": "taxes", "conflictStrategy": "error"|"skip"|"update", "dryRun": false, "rows": [ {..}, {..} ] }
OR
  Content-Type: multipart/form-data
  entity=taxes  conflictStrategy=error  file=<CSV file>
Response 200:
  { "jobId": "<uuid>", "entity": "taxes", "total": N, "imported": N, "updated": N, "skipped": N, "failed": N,
    "errors": [ { "row": <1-based index>, "message": "..." } ], "dryRun": false }
```

## Scope
- SUPER_ADMIN: may import for any clinic (row supplies `clinic_id`).
- CLINIC_ADMIN: clinic-scoped — `clinic_id` is **forced** to `kc.clinicId` on every row (rows cannot target another clinic).
- Others: `import_manage` excludes them (403).

import type { ImportEntity } from './validation';

export type ConflictStrategy = 'error' | 'skip' | 'update';
export interface RowOutcome { action: 'imported' | 'updated' | 'skipped'; }

export interface ImportAdapter {
  /** Find an existing row's id by the natural conflict key; null if none. */
  findExisting(row: any, kc: KcActor): Promise<number | null>;
  /** Insert a validated row. For non-super actors, clinic is forced to kc.clinicId (handled here). */
  insert(row: any, kc: KcActor): Promise<void>;
  /** Update an existing row (only used when conflictStrategy='update'). Optional; if absent, 'update' behaves as 'skip'. */
  update?(id: number, row: any, kc: KcActor): Promise<void>;
}

- [ ] **Step 2: Engine (`engine.ts`)**
```ts
import { randomUUID } from 'crypto';
import Papa from 'papaparse';
import { KcError } from '@/lib/kc-response';
import type { KcActor } from '@/services/billing/kc-actor';
import { rowSchemas, type ImportEntity } from './validation';
import { adapters, type ConflictStrategy } from './adapters';

export interface ImportResult {
  jobId: string; entity: ImportEntity; total: number;
  imported: number; updated: number; skipped: number; failed: number;
  errors: Array<{ row: number; message: string }>; dryRun: boolean;
}

/** Parse a CSV string into row objects (header row required). */
export function parseCsv(csv: string): Record<string, unknown>[] {
  const res = Papa.parse<Record<string, unknown>>(csv, { header: true, skipEmptyLines: true, dynamicTyping: false });
  if (res.errors.length) throw new KcError(`CSV parse error: ${res.errors[0].message} (row ${res.errors[0].row})`, 400);
  return res.data;
}

export async function runImport(
  entity: ImportEntity,
  rawRows: Record<string, unknown>[],
  opts: { conflictStrategy: ConflictStrategy; dryRun: boolean },
  kc: KcActor,
): Promise<ImportResult> {
  const adapter = adapters[entity];
  if (!adapter) throw new KcError(`Unsupported entity: ${entity}`, 400);
  const schema = rowSchemas[entity];
  const result: ImportResult = { jobId: randomUUID(), entity, total: rawRows.length, imported: 0, updated: 0, skipped: 0, failed: 0, errors: [], dryRun: opts.dryRun };

  for (let i = 0; i < rawRows.length; i++) {
    const rowNo = i + 1;
    const parsed = schema.safeParse(rawRows[i]);
    if (!parsed.success) { result.failed++; result.errors.push({ row: rowNo, message: parsed.error.issues[0]?.message ?? 'Invalid row' }); continue; }
    const row = parsed.data as any;
    try {
      const existingId = await adapter.findExisting(row, kc);
      if (existingId !== null) {
        if (opts.conflictStrategy === 'error') { result.failed++; result.errors.push({ row: rowNo, message: 'Conflict: a matching record already exists' }); continue; }
        if (opts.conflictStrategy === 'skip' || !adapter.update) { result.skipped++; continue; }
        if (!opts.dryRun) await adapter.update(existingId, row, kc);
        result.updated++; continue;
      }
      if (!opts.dryRun) await adapter.insert(row, kc);
      result.imported++;
    } catch (err) {
      result.failed++;
      result.errors.push({ row: rowNo, message: err instanceof KcError ? err.message : 'Insert failed' });
    }
  }
  return result;
}
```

```ts
import { prisma } from '@/lib/db';
import { KcError } from '@/lib/kc-response';

/** Create a wp_users row + capability meta + clinic mapping in ONE transaction. Returns wpUserId. */
export async function provisionWpUser(opts: {
  name: string; email: string; clinicId: bigint;
  capabilitySerialized: string;   // e.g. 'a:1:{s:15:"kiviCare_doctor";b:1;}'
  mappingTable: 'wp_kc_doctor_clinic_mappings' | 'wp_kc_patient_clinic_mappings';
  mappingIdCol: 'doctor_id' | 'patient_id';
  mappingHasOwner: boolean;       // doctor mapping has an `owner` column
}): Promise<number> {
  const existing = await prisma.$queryRawUnsafe<any[]>(`SELECT ID FROM wp_users WHERE user_email = ? LIMIT 1`, opts.email);
  if (existing[0]) throw new KcError('A user with this email already exists', 409);
  const username = opts.email.split('@')[0].slice(0, 60);
  const first = opts.name.split(' ')[0];
  const last = opts.name.split(' ').slice(1).join(' ') || '-';
  const placeholderHash = '!disabled-' + username.slice(0, 20);
  return prisma.$transaction(async (tx) => {
    await tx.$executeRawUnsafe(
      `INSERT INTO wp_users (user_login, user_pass, user_nicename, display_name, user_email, user_url, user_registered, user_activation_key, user_status)
       VALUES (?, ?, ?, ?, ?, '', NOW(), '', 0)`,
      username, placeholderHash, username, opts.name, opts.email);
    const idRow = await tx.$queryRawUnsafe<any[]>(`SELECT LAST_INSERT_ID() AS id`);
    const wpId = Number(idRow[0].id);
    await tx.$executeRawUnsafe(
      `INSERT INTO wp_usermeta (user_id, meta_key, meta_value) VALUES (?, 'first_name', ?), (?, 'last_name', ?), (?, 'wp_capabilities', ?), (?, 'wp_user_level', '0')`,
      wpId, first, wpId, last, wpId, opts.capabilitySerialized);
    if (opts.mappingHasOwner) {
      await tx.$executeRawUnsafe(`INSERT INTO ${opts.mappingTable} (${opts.mappingIdCol}, clinic_id, owner, created_at) VALUES (?, ?, 0, NOW())`, wpId, opts.clinicId);
    } else {
      await tx.$executeRawUnsafe(`INSERT INTO ${opts.mappingTable} (${opts.mappingIdCol}, clinic_id, created_at) VALUES (?, ?, NOW())`, wpId, opts.clinicId);
    }
    return wpId;
  });
}
```
> IMPLEMENTER: `mappingTable`/`mappingIdCol` are from a fixed internal union (never user input) so interpolating them is safe; all VALUES are bound `?`. Verify capability lengths: `kiviCare_doctor`=15 → `a:1:{s:15:"kiviCare_doctor";b:1;}`, `kiviCare_patient`=16 → `a:1:{s:16:"kiviCare_patient";b:1;}`.
