## Accepted Deviations

### 2026-06-02 - Professional ↔ Practice cardinality
One professional belongs to exactly one practice in v1. This simplifies the data model and authorization. Multi-practice membership will be addressed in a later phase when explicitly required.

### 2026-06-02 - Slot generation intersects availability with service duration
Slot grids are generated by intersecting the professional's weekly availability windows with each service's duration. A 60-min service produces different slot intervals than a 120-min service on the same availability window.

## Watchpoints (Implementation)

- Slot grid must remain non-overlapping when multiple services with different durations are assigned to the same professional
- Off-day overrides and holiday overrides must be unioned (any match = no slots)
- Status changes must propagate to slot API within 5 seconds (SC-005)
- All times stored in UTC; practice timezone conversion at API edge
- Holiday blocking (BR-05.05) deferred to feature 013 — slot generation treats holidays as no-ops until holiday entity exists
- PENDING sessions block slot generation (same as BOOKED); only COMPLETED, CANCELLED, REJECTED do not block
- Availability windows cannot overlap on the same day-of-week (FR-015)
